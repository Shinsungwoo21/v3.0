# Bedrock 챗봇 고도화 계획 (제안)

현재 구현된 기본 챗봇 기능을 넘어, 에이전트를 더욱 강력하게 만들 수 있는 기능 추가 아이디어입니다.

## 1. 멀티모달(Mulitmodal) 지원
- **기능**: 사용자가 이미지를 업로드하고, 이미지에 대해 질문할 수 있는 기능을 추가합니다.
- **적용**: Claude 3.5 Sonnet 및 Nova 모델은 비전(Vision) 기능을 지원하므로, `converse_stream` API의 `content` 파라미터에 이미지를 포함시켜 구현할 수 있습니다.
- **UI**: `st.file_uploader`를 채팅창 근처에 배치.

## 2. RAG (검색 증강 생성) 연동
- **기능**: 사용자가 업로드한 PDF, 텍스트 문서 등을 기반으로 답변하는 기능입니다.
- **구현 방법**:
    1. **Simple RAG**: 사용자가 업로드한 파일을 텍스트로 추출하여 프롬프트 컨텍스트에 포함.
    2. **Bedrock Knowledge Base**: AWS Bedrock의 Knowledge Base와 연동하여 대규모 문서 검색 및 답변 생성.

## 3. 시스템 프롬프트 커스터마이징
- **기능**: 사용자가 사이드바에서 직접 시스템 프롬프트(페르소나)를 수정할 수 있는 기능을 제공합니다.
- **활용**: "번역가 모드", "코딩 튜터 모드" 등 다양한 역할을 즉석에서 부여할 수 있습니다.

## 4. 대화 내보내기 (Export)
- **기능**: 현재 대화 내용을 Markdown 또는 JSON 파일로 다운로드하는 기능입니다.
- **UI**: 사이드바에 "대화 저장하기" 버튼 추가. `st.download_button` 활용.

## 5. 고급 추론 파라미터 제어
- **기능**: Temperature(창의성), Top-P, Max Tokens 등의 값을 사이드바 슬라이더로 조절할 수 있게 합니다.
- **활용**: 정교한 코딩 작업 시 Temperature를 낮추거나, 브레인스토밍 시 높이는 등 유연한 사용 가능.

## 6. 음성 인터페이스 (Voice I/O)
- **기능**: 마이크로 질문하고(STT), 답변을 음성으로 읽어주는(TTS) 기능.
- **구현**: Streamlit의 음성 녹음 컴포넌트와 AWS Transcribe/Polly (또는 OpenAI Whisper) 연동.
